# 测试数据使用说明

## 初始化测试数据

### 方法1：通过小程序首页入口

1. 打开微信开发者工具
2. 编译并运行小程序
3. 在首页底部找到"开发调试入口"区域
4. 点击"初始化测试数据"按钮
5. 等待数据初始化完成

### 方法2：直接访问页面

1. 在微信开发者工具中输入路径：`pages/init-data/init-data`
2. 点击"开始初始化"按钮
3. 等待数据创建完成

## 初始化的数据内容

初始化完成后，数据库将包含以下测试数据：

- **一级分类**：4个
  - 广告风格
  - 实景风格
  - 人物风格
  - 动漫风格

- **二级分类**：10个
  - 广告风格下：大牌广告、港风招贴、野广告风格
  - 实景风格下：白天场景、夜晚场景、静物写真
  - 动漫风格下：日本漫画、3D风格、欧美风格、国风

- **模板**：13个
  - 每个二级分类下都有对应的模板
  - 包含封面图（使用占位图服务）
  - 支持瀑布流3列布局显示

- **产品素材**：36个
  - 覆盖6个行业：美妆、服装、餐饮、教育、母婴、数码
  - 每个行业6个产品
  - 包含产品图片（使用占位图服务）

- **行业标签**：6个
  - 美妆、服装、餐饮、教育、母婴、数码

- **模板字段配置**：4个
  - 包含文本、文本域、选择等字段类型

- **定价规则**：4条
  - 包含图片和视频的不同定价方案

## 测试页面

初始化完成后，可以测试以下页面：

1. **模板列表页** (`pages/template-list/template-list`)
   - 顶部显示"Step1, 选择风格模板"
   - 4个一级分类Tab
   - 瀑布流3列布局
   - 每个模板卡片显示：封面图、标题、分类名称

2. **产品选择页** (`pages/product-select/product-select`)
   - 顶部显示"Step2, 选择产品"
   - 瀑布流3列布局
   - 每个产品卡片显示：图片、名称
   - 分页加载（每次30条）

## 注意事项

1. **首次使用**：必须先初始化数据，否则页面会显示"暂无模板"
2. **重复初始化**：可以多次点击初始化，不会产生重复数据（取决于云函数实现）
3. **占位图**：当前使用 via.placeholder.com 服务生成测试图片
4. **云函数部署**：如果 initData 云函数未部署，需要在微信开发者工具中右键 `cloudfunctions/initData` 文件夹，选择"上传并部署：云端安装依赖"

## 问题排查

### 点击初始化没有反应
- 检查云函数是否已部署
- 查看控制台是否有错误信息
- 确认云开发环境已正确配置

### 数据初始化成功但页面不显示
- 检查数据库集合是否正确创建
- 查看 page 的 loadTemplates/loadProducts 方法是否有错误
- 确认查询条件是否正确

### 图片加载失败
- 占位图服务可能不稳定，可以替换为其他图片URL
- 检查网络连接
- 确认图片URL格式正确
